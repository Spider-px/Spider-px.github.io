#前置知识：

1.传统访问：用户访问域名–>解析服务器IP–>访问目标主机

2.普通CDN：用户访问域名–>CDN节点–>真实服务器IP–>访问目标主机

3.带WAF的CDN：用户访问域名–>CDN节点（WAF）–>真实IP–>访问目标主机

常见方法：

子域名，邮件系统，国外访问，证书查询，APP抓包，网络空间

通过漏洞或泄露获取，扫全网，以量打量，第三方接口查询等

1. 查询历史DNS记录：通过查询历史DNS记录，可以找到使用CDN前的IP地址。常用网站有DNSDB、微步在线、Netcraft、ViewDNS等。就是网站可能之前没有开启cdn，就有真实的dns解析记录。

2. 查询子域名：一些网站可能只对主站或流量大的子站点使用CDN。通过查询子域名对应的IP地址，可以辅助找到网站的真实IP。常用工具包括微步在线、Dnsdb查询法、Google搜索等。

  1. 网络空间引擎搜索：利用网络空间搜索引擎（如Shodan、Fofa、ZoomEye）搜索特定关键词或特征，可以找到使用CDN的网站的真实IP地址。

4. SSL证书：扫描互联网获取SSL证书，进而找到服务器的真实IP。Censys是一个用于搜索联网设备信息并扫描整个互联网的强大工具。

:::info

  1. CA颁发SSL证书时强制存入公开CT日志，全网可查证书域名、SAN域名。

  2. 工具批量扫描全网443端口，抓取每台IP返回的证书指纹/内容。

  3. 建立映射：IP ↔ 证书指纹 ↔ 目标域名。

  4. 匹配到相同证书，即可反推出站点真实源站IP。 5. 历史证书、未接入CDN的子域名证书，也能挖出过往/后端真实IP。

:::

  1. HTTP标头：通过比较HTTP标头来查找原始服务器。例如，使用SecurityTrails平台搜索特定HTTP标头。

  2. 网站源代码：浏览网站源代码，寻找独特的代码片段或隐藏的IP地址信息。

  3. 国外主机解析域名：由于一些CDN服务在国外地区可能无法提供完整的保护，使用国外的主机直接访问目标网站可能会获取到真实IP地址。

8. 遗留文件：如phpinfo页面泄露，可能会显示服务器的外网IP地址。

9. 漏洞探针：利用网站上的漏洞（如SSRF漏洞）让VPS获取对方反向连接的IP地址。

10. 网站邮件订阅：查看RSS邮件订阅的邮件源码，通常包含服务器的真实IP地址。

  1. 全网扫描：使用Zmap、Masscan等工具对整个互联网进行扫描，针对扫描结果进行关键字查找，获取网站真实IP。

  2. 解码F5 LTM负载均衡器：当服务器使用F5 LTM做负载均衡时，通过对set-cookie关键字的解码，可以获取到真实IP地址。

  3. 利用MX记录：如果网站在与web相同的服务器和IP上托管自己的邮件服务器，那么原始服务器IP将在MX记录中。

  4. 利用favicon.ico：查看网站的favicon.ico文件，通过其唯一的hash值进行识别，有时可以找到与网站相关的真实IP地址。

15. 配置错误：检查CDN的配置细节，有时小小的配置错误就可能导致CDN防护被绕过。例如，CDN只配置了www域名，而直接访问非www域名可能获取到真实IP。

16. 协议差异：如果站点同时支持http和https访问（前置条件，80端口开放），而CDN只配置了https协议，那么访问http可能获取到真实IP。

  1. 利用网站功能：通过网站提供的某些功能（如邮箱注册、找回密码等），让网站主动暴露真实IP地址。

  2. 搜索引擎查询：利用搜索引擎的特定查询语法（如site:和inurl:），结合目标网站的信息，可能找到与网站相关的真实IP地址。

  3. 子域名挖掘机：输入域名即可基于字典挖掘子域名，通过查找子域名的IP地址来辅助找到网站的真实IP。

  4. IP库查询：使用IP库（如纯真数据库）来查询IP段，结合网站特征进行爆破，可能找到真实IP地址。

#某应用-CDN绕过-主动漏洞&遗留文件&邮件系统

1、漏洞如：SSRF RCE等

利用网站上的漏洞（如SSRF漏洞）让VPS获取对方反向连接的IP地址。就是利用对方业务上的漏洞来让对方服务器主动交出ip，我们在公网上通过nc等工具监听自己端口，通过可以加载外部资源的功能点访问攻击机，那就得到真实ip了

2、遗留文件：phpinfo类似功能

如phpinfo页面泄露，可能会显示服务器的外网IP地址。这种就是靠漏洞信息泄露导致，有可能也会显示内网ip，自己判别

3、邮箱系统：

查看RSS邮件订阅的邮件源码，通常包含服务器的真实IP地址。这种情况获取的概率较低，第一种情况：第三方代发，就是可能用了其他企业专门的邮箱系统，所以自然获取不了。第二种情况：邮件服务器和目标关系不大，如果两个都不在一个服务器上自然也是不行。第三种情况：自己邮箱的问题，比如你用qq邮箱这种，它可能不是直达，是先发给qq然后再给你，所以看到的ip是qq的，但是这种有办法解决，就是搭建自己的邮件系统。

常见的邮件触发点有：

1、RSS订阅

2、邮箱注册、激活处

3、邮箱找回密码处

4、产品更新的邮件推送

5、某业务执行后发送的邮件通知

6、员工邮箱、邮件管理平台等入口处的忘记密码
