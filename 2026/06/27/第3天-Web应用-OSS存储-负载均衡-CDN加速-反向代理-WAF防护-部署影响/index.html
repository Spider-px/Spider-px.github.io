#CDN

原理：内容分发服务，旨在提高访问速度

影响：隐藏真实源IP，导致对目标IP测试错误

演示：阿里云备案域名全局CDN加速服务

Windows2016 + BT宝塔面板 + CDN服务

#

CDN核心就是“就近加速、减轻源站带宽压力”，针对静态内容做缓存，用户访问的是CDN边缘节点的缓存数据，99%的情况不会触达源站

怎么判断开没开CDN：用网上的超级ping网站，看看多地ping的ip结果一不一致，一致就是没开，不一致就是开了

#反向代理

正代理为客户端服务,客户端主动建立代理访问目标（不代理不可达），客户端配置的。

反向代理为服务端服务,服务端主动转发数据给可访问地址（不主动不可达），客户端不知道反向代理存在。

原理：通过网络反向代理转发真实服务达到访问目的

影响：访问目标只是一个代理，非真实应用服务器

注意：正向代理和反向代理都是解决访问不可达的问题，但由于反向代理中多出一个可以重定向解析的功能操作，导致反代理出的站点指向和真实应用毫无关系！

演示：Nginx反向代理配置

Windows2016 + BT宝塔面板 + Nginx

反向代理：就很类似一个重定向操作，就比如你访问一个网站他开了反向代理，目标url是baidu.com，服务器给浏览器返回的页面是百度的东西，但是url全程是没变的，重定向url会变成baidu.com。反向代理服务器就是它去取来数据，然后给用户，全程url还是反向代理服务器的url

#负载均衡

负载均衡（Load Balancing） 是一种网络与服务器架构技术，核心目标是将分布式的请求/任务均匀分配到多个后端服务器（或节点）上，避免单一节点过载，提升系统的吞吐量、可用性与稳定性。

影响：有多个服务器加载服务，测试过程中存在多个目标情况

有没有人看完有一个疑问：cdn和负载均衡怎么判断呢，都是访问一个网站，可能ip不一样。用超级ping ip多的是cdn，少的是负载均衡（因为负载均衡是核心服务器，cdn就是缓存），最关键我觉得就是，我们多访问几次，负载均衡很可能ip会变，cdn不变
